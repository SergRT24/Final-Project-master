History Of CSS

Before Cascading Style Sheets, commonly known as CSS, there wasn't much that could be done to change the design of a web page. The term cascading comes from the ability to comvine multiple CSS fles to determine the style for one page. As many people started using HTML, as it was groundbreaking, the demand grew for more design capabilites, which would allow developers to control how web documents looked.

Although it took 3 years for any browser to come close to fully implementing CSS, August 1996 Microsoft Internet Explorer became the first browser to support CSS. Netscape followed suit in supporting CSS, but also implemented an alternative Javascript Style Sheets, which were never fully completed, and are now deprecated. To this day, there are differences in the way CSS is implemented in different browsers, leading developers to use hacks to make web pages look consistent in different browsers.

History of HTML

The history of HTMl, also known as Hypertext Markup Language, is pretty strange, and pretty interesting. From its start as an online subset, through political maneuverings of large browser companies, to its current state of compatibility. The coding language has endured a storm of growth, abuse, and groundbreaking innovations.

As the battle for internet superiority continues amongst various practical and complicated text languages, there is one topic that they are heavily focusing on. That topic is functionality. As more and more people are learning to use computers in a society that is now driven by the ability to advance technology. Companies like Microsoft and Netscape are now showing off W3C compliance as a critical marketing advantage. Which is causing panic for the future of HTML. Only time will tell if HTML will be as prominent in the future as it has been in the past.
